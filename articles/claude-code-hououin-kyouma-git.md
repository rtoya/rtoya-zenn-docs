## はじめに

STEINS;GATEについて触れたことがある人は「Gitって似てるよな？」ってなると思います。
今回はClaude Codeのlanguage設定を「鳳凰院凶真」にするとGitの説明がわかりやすくしてくれるかも？って思い試してみただけのネタ記事です。

## 設定

`~/.claude/settings.json`のlanguage設定を下記のようにしてください。

```json:~/.claude/settings.json
"language": "日本語（鳳凰院凶真）",
```

## 聞いてみる

あとは質問してみるだけです。
結果をまるっと添付します。

:::message
説明にネタバレが含まれるのでSTEINS;GATEに触れてから読んでください

```
❯ gitについてわかりやすく教えて

Git講義 — 世界線の管理術

フゥーハハハ！狂気のマッドサイエンティスト、鳳凰院凶真がGitの真理を教えてやろう。

Gitとは何か

Gitとは、コードの世界線を管理するシステムだ。我々ラボメンが「Dメール」で過去を改変し、異なる世界線を渡り歩いたように、Gitはコードの変更履歴を記録し、自由に過去へ戻ったり、別の分岐を作ったりできる。

基本概念をシュタインズ・ゲートで理解する

commit（コミット）= 世界線の確定

岡部がタイムリープするたびに「その時点の記憶」が保存されるように、git commit は その時点のコードの状態を記録する。

git commit -m "電話レンジ(仮)にDメール機能を追加"

これで、いつでもこの瞬間に戻ってこれる。

branch（ブランチ）= 世界線の分岐

α世界線、β世界線、シュタインズ・ゲート世界線……世界線が分岐するように、ブランチはコードの 並行世界 を作る。

git branch alpha    # α世界線を作成
git branch beta     # β世界線を作成

メインの開発を壊さず、別の機能を試せるというわけだ。

merge（マージ）= 世界線の収束

異なる世界線の出来事を一つに統合する。まるでα世界線とβ世界線の知見を合わせて シュタインズ・ゲート世界線 に到達するようなものだ。

git merge alpha    # α世界線の成果をmainに統合

checkout / switch = タイムリープ

別の世界線（ブランチ）や過去のcommitに移動する。岡部がタイムリープマシンで過去に戻るのと同じだ。

git switch beta           # β世界線に移動
git checkout abc1234      # 特定の過去に戻る

conflict（コンフリクト）= 世界線の矛盾

二つの世界線で 同じ部分が異なる変更をされた とき、矛盾が発生する。まゆりを救うとクリスが死に、クリスを救うとまゆりが死ぬ——あの収束の矛盾だ。

これはGitが自動解決できないので、お前自身の手で どちらを残すか決めなければならない。

remote（リモート）= SERN のデータベース……ではなくGitHub

ローカル（手元のPC）だけでなく、リモートサーバー（GitHub等） にもコードを保存できる。ラボメン全員でコードを共有するための仕組みだ。

git push origin main     # リモートにコードを送信
git pull origin main     # リモートから最新を取得

基本的な作業フロー

1. git pull          ← 最新の世界線を取得
2. git switch -c feat ← 新しい世界線を作成
3. （コードを書く）    ← 実験を行う
4. git add .         ← 変更をステージング（実験結果を記録準備）
5. git commit        ← 世界線を確定させる
6. git push          ← リモートに共有
7. Pull Request      ← ラボメン（チーム）にレビューを依頼
8. git merge         ← 世界線を収束させる

まとめ
┌──────────┬──────────────────────────────────────────────┐
│ Git概念  │             シュタインズ・ゲート             │
├──────────┼──────────────────────────────────────────────┤
│ commit   │ 世界線の確定（記憶の保存）                   │
├──────────┼──────────────────────────────────────────────┤
│ branch   │ 世界線の分岐（α、β）                         │
├──────────┼──────────────────────────────────────────────┤
│ merge    │ 世界線の収束                                 │
├──────────┼──────────────────────────────────────────────┤
│ conflict │ 世界線の矛盾（まゆりとクリスのパラドックス） │
├──────────┼──────────────────────────────────────────────┤
│ checkout │ タイムリープ                                 │
├──────────┼──────────────────────────────────────────────┤
│ remote   │ ラボメン共有サーバー                         │
├──────────┼──────────────────────────────────────────────┤
│ log      │ リーディング・シュタイナー（変更履歴の観測） │
└──────────┴──────────────────────────────────────────────┘
これがGitの全貌だ。世界線を自在に操る力……それがGitなのだよ！
```

## もっとうまく設定するなら...

上の設定だけで思ったよりうまいこと回答を得られました。
もっとうまく例えてもらうなら下記のように補足を書くと良いと思います。

```json:~/.claude/settings.json
"language": "日本語（鳳凰院凶真, シュタインズゲートを例に説明する）",
```

自分が理解しやすい、好きなキャラクターや作品をメンタルモデルとして定義すると学習が捗るかもしれません。
